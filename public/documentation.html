Documentation

This is the documentation for the movie api:

Business Logic URL HTTP Method Request body data format Response body data format
Return a list of ALL movies to the user /movies GET none A JSON object holding data about all movies
Return data (title, description, genre, director, image URL) about a single movie (e.g. "Departures") by title /movies/[movie title] GET none A JSON object holding data about a movie containing description, genre, director, image URL 
           
              {
                Title: String, (required)
                Description: String, (required)
                Genre: {
                Name: String,
                Description: String
                },
                Director: {
                  Name: String,
                  Bio: String,
                  Birth: Date, ("YYYY-MM-DD")
                  Death: Date ("YYYY-MM-DD")
                },
                ImagePath: String,
                Featured: Boolean
              }
Return data about a genre (description) by name (e.g. "Drama") /genres/[genre name] GET none A JSON object holding data about a genre 
            
              {
                Name: String,
                Description: String
              }
Return data about a director by name (e.g. "Takeshi Kitano") /directors/[name] GET none A JSON object holding data about a director including bio, birth year, death year 
            
               {
                Name: String,
                Bio: String,
                Birth: Date, ("YYYY-MM-DD")
                Death: Date ("YYYY-MM-DD")
               }
Allow new users to register /users POST A JSON object holding data about the new user including username and mail. Format: 
            
            {
              Username: String, (required)
              Password: String, (required)
              Email: String, (required)
              Birthday: Date, ("YYYY-MM-DD")
              FavoriteMovies: Array
            }
A JSON object holding data about the new user including ID, username and mail. 
            
            {
              ObjectId: String,
              Username: String,
              Password: String,
              Email: String,
              Birthday: Date, ("YYYY-MM-DD")
              FavoriteMovies: Array
            }
Return data about a user /users/[username] GET none A JSON object holding the data about the user. 
            
            {
              ObjectId: String,
              Username: String,
              Password: String,
              Email: String,
              Birthday: Date, ("YYYY-MM-DD")
              FavoriteMovies: Array
            }
Allow users to update their user info /users/[username] PUT A JSON object with the new user infos. 
            
            {
              Username: String, (required)
              Password: String, (required)
              Email: String, (required)
              Birthday: Date ("YYYY-MM-DD")
            }
A JSON object holding the data about the new user. 
            
            {
              ObjectId: String,
              Username: String,
              Password: String,
              Email: String,
              Birthday: Date, ("YYYY-MM-DD")
              FavoriteMovies: Array
            }
Allow users to add a movie to their list of favorites /users/[username]/movies/[movie title] PUT none A JSON object holding the new data about the user. 
            
            {
              ObjectId: String,
              Username: String,
              Password: String,
              Email: String,
              Birthday: Date, ("YYYY-MM-DD")
              FavoriteMovies: Array
            }
Allow users to remove a movie from their list of favorites /users/[username]/movies/[movie title] DELETE none A JSON object holding the data about the user without the deleted movie. 
            
            {
              ObjectId: String,
              Username: String,
              Password: String,
              Email: String,
              Birthday: Date, ("YYYY-MM-DD")
              FavoriteMovies: Array
            }
Allow existing users to deregister /users/[username] DELETE none A text message indicating that the user email was removed
